📦 安装部署

baka Bot 采用前后端分离架构，需要同时部署 NoneBot2 前端和 NapCat 后端。

系统要求

  • Linux 服务器（推荐 Ubuntu/Debian）
  • Python 3.10+
  • 一个 QQ 小号
  • Moonshot API Key

一、前端部署（NoneBot2）

更新系统软件包

bash
apt update

安装 Python 环境和必要工具

bash
apt install python3-venv python3-pip git screen unzip -y

下载 baka Bot

bash
# 从 GitHub 克隆
git clone https://github.com/zther-zips/baka-Bot.git

# 或者本地上传后解压
unzip baka-Bot.zip

创建并激活虚拟环境

bash
cd baka-Bot
python3 -m venv venv
source venv/bin/activate

安装依赖

bash
# 安装 NoneBot2 和 OneBot 适配器
pip install nonebot2[fastapi] nonebot-adapter-onebot

# 安装项目依赖
pip install .

# 安装其他依赖（使用清华镜像加速）
pip install openai aiofiles -i https://pypi.tuna.tsinghua.edu.cn/simple

使用 Screen 后台运行

bash
# 先退出虚拟环境
deactivate

# 创建 screen 会话
screen -S baka

# 进入项目目录并激活环境
cd /root/baka-Bot
source venv/bin/activate

# 启动 bot
python3 bot.py

按 Ctrl+A 再按 D 分离会话，保持后台运行。

💡
Screen 常用命令

screen -ls - 查看所有会话 | screen -r baka - 回到 baka 会话 | screen -wipe - 清理死会话

二、后端部署（NapCat）

安装 NapCat

bash
curl -o napcat.sh \n  https://nclatest.znin.net/NapNeko/NapCat-Installer/main/script/install.sh \n  && bash napcat.sh --tui

或使用 CLI 模式安装：bash napcat.sh --cli y

获取 WebUI Token

bash
cat /root/Napcat/opt/QQ/resources/app/app_launcher/napcat/config/webui.json

记录下其中的 token，用于登录 WebUI。

配置端口映射（如需要）

bash
# 本地 SSH 映射到服务器端口
ssh -L 6099:127.0.0.1:6099 root@你的服务器IP

启动 NapCat

bash
# 创建 screen 会话启动 NapCat
screen -S napcat

# 启动 NapCat（替换为你的 QQ 号）
xvfb-run -a /root/Napcat/opt/QQ/qq --no-sandbox -q 你的QQ号

# 按 Ctrl+A 再按 D 分离会话

配置 WebSocket 连接

打开浏览器访问 WebUI：

  • 本地映射：http://127.0.0.1:6099
  • 直接访问：http://服务器IP:6099/webui/

输入 token 登录后，在网络设置中：

  1. 新建 WebSocket 客户端
  2. 启用连接
  3. 名称填：baka
  4. URL 填：ws://127.0.0.1:8080/onebot/v11/ws
⚠️
注意

请确保 8080 端口未被占用，这是 NoneBot2 默认的 WebSocket 端口。如果需要修改，请在 .env 文件中配置。

🚀 使用指南

部署完成后，你就可以开始和 baka 聊天啦～

启动顺序

启动后端（NapCat）

bash
screen -dmS napcat bash -c "xvfb-run -a /root/Napcat/opt/QQ/qq --no-sandbox -q 你的QQ号"

启动前端（baka Bot）

bash
screen -S baka
cd /root/baka
source venv/bin/activate
python3 bot.py

日常管理命令

操作命令
查看所有会话screen -ls
进入 baka 会话screen -r baka
进入 napcat 会话screen -r napcat
分离会话（后台保持）Ctrl+A 然后 D
结束运行Ctrl+C
停止 napcat 会话screen -S napcat -X quit
清理死会话screen -wipe

与 baka 互动

  • 提及回复：在消息中包含 "baka" 或 @机器人，即可触发 AI 回复
  • 私聊：直接发送消息给机器人，无需特殊格式
  • 戳一戳：双击机器人头像戳一戳，会有可爱回应

⚙️ 配置说明

baka Bot 的配置主要通过环境变量和配置文件进行。

环境变量配置 (.env)

env
# 运行环境
ENVIRONMENT=prod

# Moonshot API Key（必填）
MOONSHOT_API_KEY=your-api-key-here

# 超级用户 QQ 号（可选）
SUPERUSERS=["123456789"]

# 命令起始字符（可选）
COMMAND_START=["/"]

获取 Moonshot API Key

  1. 访问 Moonshot 开放平台
  2. 注册并登录账号
  3. 进入「API Key 管理」页面
  4. 创建新的 API Key
  5. 将 Key 复制到 .env 文件中
🔒
安全提示

请勿将包含 API Key 的 .env 文件提交到 Git 仓库！建议在 .gitignore 中添加 .env。

✨ 功能介绍

🤖 AI 智能对话

基于 Moonshot AI (Kimi) 实现，支持多轮对话上下文记忆。baka 会根据聊天历史理解语境，给出自然流畅的回复。

  • 支持上下文记忆（最多保存最近 10 条对话）
  • 自动识别提及和 @ 消息
  • 私聊无需触发词，直接对话
  • 群聊需要包含 "baka" 或 @机器人

💕 好感度系统

每个用户都有独立的好感度值，baka 会根据好感度调整回复风格：

  • 好感度越高，回复越亲昵撒娇
  • 数据持久化保存，重启不丢失
  • 可以手动修改 favorability.json 调整好感度

👆 戳一戳响应

支持 QQ 戳一戳功能，当用户戳 baka 时会有可爱的回应。

🖼️ 图片保存

自动检测并保存聊天中的图片到本地，方便后续管理和查看。

📝 日志记录

完整的日志系统，记录所有聊天内容，方便调试和回顾。
